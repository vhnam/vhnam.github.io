Hướng dẫn thao tác với repository trên GitHub

Chức năng chính của GitHub dùng để lưu trữ source code. Bài viết này, mình sẽ giúp các bạn thao tác với một repository trên GitHub. Có một số vấn đề trong bài viết này, mình sẽ sử dụng TortoiseGit để thao tác với GitHub và sử dụng giao thức SSH để truyền dữ liệu.

Tạo SSH Key

Bạn mở PuTTYgen để bắt đầu tạo SSH Key. Lưu ý là mã được tạo ra bằng thuật toán RSA. Do đó, bạn sẽ cần lưu 2 khóa public và private.

Tiếp theo, bạn nhấn nút Generate và di chuyển con chuột trong vùng được khoanh màu đỏ để tạo SSH Key.

Sau khi hoàn tất, bạn nhấn Save private key để lưu khóa private. Tiếp theo, bạn copy đoạn trong khung được khoanh tròn để lưu lại khóa public.

TortoiseGit

Đầu tiên, bạn phải vào trang http://git-scm.com/download/win để tải Git về và cài đặt.

Tiếp theo, vào trang https://code.google.com/p/tortoisegit/wiki/Download để tải TortoiseGit rồi cài. Sau khi cài xong, bạn nhấn chuột phải sẽ ra như hình bên dưới.

Thêm SSH Key vào GitHub

Bạn vào phần Settings trên GitHub. Nhìn sang cột bên trái, chọn SSH Keys.

Nhấn nút Add Key, điền Title và copy Public Key vào khung Key.

Nhấn nút Add key để nó điều hướng sang trang khác. Xác nhận password là xong.

Tạo repository

Sau khi tạo xong SSH Key, cài đặt Git và TortoiseGit xong, chúng ta sẽ tạo một repository trên GitHub.

Ta vào trang chủ trên GitHub, chọn tab Repositories, nhấn nút New.

Bạn điền đẩy đủ thông tin bao gồm tên của repo, mô tả (description), public hay private, có cần file README hay không, chịu ảnh hưởng của bản quyền mã nguồn mở nào,...

Và đây là kết quả.

Git

Mình chỉ giới thiệu Git theo như những gì mình hiểu về nó thôi. Tư tưởng của nó là sản phẩm của bạn có thể có nhiều nhánh để phát triển. Trong đó, nhánh master là nhánh hoàn thiện nhất của dự án.

Ví dụ, dự án của bạn gồm có 3 người làm là Tủn, Tồ và Tèo. Nếu làm theo cách thông thường, Tủn làm xong việc mới, Tồ mới làm tiếp được. Tèo muốn thay đổi mã nguồn thì Tủn và Tồ thở oxy. Làm theo tư tưởng trên. Mỗi người sẽ tạo một nhánh (branch). Mỗi nhánh tương đương với một chức năng trong dự án. Ai làm xong chức năng nào thì merge vào nhánh master.

Trong quá trình phát triển, người ta sẽ chia ra từng chặng đường nhỏ. Trong mỗi chặng đường, công việc bao gồm phân tích - phác thảo - thiết kế - kiểm thử. Vòng lặp sẽ liên tục cho tới khi hoàn thành dự án sẽ kết thúc.

Thực ra, Git là công cụ implement triết lý Agile trong phương pháp lập trình linh hoạt.

Phát triển

Nhìn xuống góc bên phải, chọn vào SSH để lấy được URL cho việc clone bằng giao thức SSH. Sau đó, vào thư mục cần lưu trên máy tính, chuột phải chọn Git clone rồi đánh dấu vào ô Load Putty Key, chọn đường dẫn tới nơi chứa private key có định dạng *.ppk. Nếu bạn chọn HTTPS thì KHÔNG cần chọn vào ô đó. Lưu ý là URL của 2 giao thức này khác nhau.

Mặc định, bạn sẽ thao tác với nhánh master. Để tạo một nhánh khác, bạn chuột phải vào repo lưu trên máy, chọn TortoiseGit -> Create Branch.

Sau đó, điền tên nhánh cần tạo, nhánh mới rẽ từ nhánh nào, muốn chuyển sang nhánh mới liền sau khi tạo hay không,...

Chuột phải vào repo lưu trên máy để kiểm tra đã chuyển sang nhánh mới hay chưa.
